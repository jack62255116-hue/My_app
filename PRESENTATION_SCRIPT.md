## 0:35~1:15 - 사용자 시나리오 (앱 사용 흐름)

**[타이머: 1:25]**

**실제 사용하면 이렇습니다:**

1단계: 사용자가 + 버튼을 누릅니다.

2단계: AI가 지난 30일 기록을 분석합니다.
"아, 이 사람은 매일 5km씩 달렸네"

3단계: AI가 제안합니다.
"오늘도 운동 5km 할 거 같아?"

4단계: 사용자는 3가지 중 선택합니다.
- "응" → 바로 저장
- "3km로 해줘" → 수정해서 저장  
- "건너뛸래" → 직접 입력

전체 시간? **2~3초입니다.**

## 1:15~1:50 - 기술 아키텍처 + 선택 이유

**[타이머: 0:45]**

**3계층 구조:**

**[위층 - UI]**
Flutter로 만들어서 iOS, Android 동시 지원

**[중층 - AI 뇌]**
Pattern Analyzer가 패턴을 학습
Riverpod으로 빠른 반응

**[아래층 - 저장소]**
- 왼쪽: SQLite (휴대폰 내부, 프라이버시 보호)
- 오른쪽: Claude API (최고 성능의 AI)

**[왜 이 기술?]**
- Flutter: iOS/Android 동시 개발 가능
- SQLite: 빠르고 안전함
- Claude Sonnet: 가장 똑똑한 AI
